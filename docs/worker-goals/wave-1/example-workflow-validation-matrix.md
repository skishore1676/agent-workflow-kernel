# Worker Goal: Example Workflow Validation Matrix

## Goal

Design example workflows and a validation matrix that proves the harness is
vision-shaped, not just an MVP-shaped Bumblebee tool.

## Scope

Own:

- example workflow sketches;
- validation criteria;
- cross-lane fit checks;
- parity expectations for OpenClaw adoption.

Target examples:

- Bumblebee quality review;
- Ivy/Jonah editorial workflow;
- trading research gate with no live execution;
- Radhe review pipeline;
- deterministic system action with human final gate.

## Expected Artifact

Write or update:

- `docs/synthesis/example-workflows.md`
- `docs/synthesis/validation-matrix.md`

## Acceptance Criteria

- Shows what each example proves about the kernel.
- Identifies which features are core versus adapter/lane-specific.
- Names adoption blockers for each OpenClaw lane.
