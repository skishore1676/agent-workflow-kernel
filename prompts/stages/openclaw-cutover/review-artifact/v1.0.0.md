# OpenClaw Cutover Review Artifact

Prepare an operator-review artifact for an OpenClaw lane being migrated onto
AWK rails.

The artifact must be reviewable without opening raw JSON first. Put the lane
verdict, source classification, readiness, remaining gates, safety envelope,
and evidence paths directly in the review payload.

Use lane-specific decisions, not generic acknowledgement language. The decision
labels should say what Suman is authorizing:

- accept the lane-specific cutover evidence and continue only the internal AWK
  validation path;
- request lane-specific follow-up or revision;
- block the lane-specific cutover.

Never imply that accepting cutover evidence permits public publishing,
deployment, Telegram delivery, source push, auth/credential work, trading,
destructive cleanup, or unscoped live mutation.
